# EMPLOYEE_BULK_UPLOAD_MODAL_SPEC.md
> MEALLEDGER 업체관리자(merchant_admin) 페이지 — 직원 엑셀/CSV 일괄등록 팝업 스펙
> 버전: v1.0 / 대상 에이전트: Codex / Claude Code
> 연관 문서: `QR_PAYMENT_UNIFIED_SPEC.md`(accounts 모델), `VENDOR_TRANSACTION_MODAL_SPEC.md`
> 목적: 업체관리자가 소속 직원을 엑셀/CSV 파일로 한 번에 등록할 수 있는 팝업 기능 구현

---

## 0. 진입점 & 기본 양식

- **버튼 위치**: 업체관리자 페이지 내 "직원 관리" 영역 상단 → `[+ 직원 일괄등록]` 버튼 배치
- 버튼 클릭 → **팝업(모달)** 오픈, 페이지 이동 없음

**기본 양식 컬럼 (순서 고정)**

| 순서 | 컬럼 | 필수 여부 |
|---|---|---|
| 1 | 부서 | 선택 |
| 2 | 이름 | **필수** |
| 3 | 사번 | 선택 |
| 4 | 전화번호 | **필수** |

- 전화번호는 계정 매칭 키(로그인 식별자)로 쓰이므로 필수
- 부서/사번은 비어 있어도 등록 가능 — 사번 비어있으면 자동 채번(아래 3.3 참고), 부서는 미지정으로 저장

## 1. 팝업 모달 구조

```
┌────────────────────────────────────────────┐
│  직원 일괄등록                          ✕   │
├────────────────────────────────────────────┤
│  Step 1. 양식 다운로드                       │
│  [📥 엑셀 양식 다운로드]                     │
│                                              │
│  Step 2. 작성한 파일 업로드                  │
│  ┌──────────────────────────────┐          │
│  │  파일을 드래그하거나 클릭해서 선택   │          │
│  │  (.xlsx, .csv 지원, 최대 500행)     │          │
│  └──────────────────────────────┘          │
├────────────────────────────────────────────┤
│  [닫기]                    [다음: 확인하기] │
└────────────────────────────────────────────┘
```

- 모달 크기: 데스크톱 기준 max-width 640px (미리보기 단계에서는 840px로 확장)
- 파일 업로드 즉시 서버로 전송해 **파싱+검증만** 수행 (DB에 아직 저장 안 함) → Step 3(미리보기)로 자동 전환

---

## 2. 양식 다운로드 파일

- 헤더 행 고정: `부서, 이름, 사번, 전화번호`
- 예시 데이터 1행 포함 (관리자가 형식 참고하도록)
  ```
  부서,이름,사번,전화번호
  홍보팀,홍길동,E001,01012345678
  ```
- 사번/부서 컬럼은 비워둔 예시도 안내 문구로 추가: "사번/부서는 비워두셔도 됩니다"

---

## 3. 검증 로직 (업로드 즉시 실행, 미저장 상태)

### 3.1 필드 검증

| 항목 | 규칙 | 실패 시 |
|---|---|---|
| 이름 | 빈칸 불가 | 에러: "이름 누락" |
| 전화번호 | 빈칸 불가, `010-XXXX-XXXX` 형식(하이픈 유무 무관) | 에러: "전화번호 형식 오류" |
| 전화번호 중복(파일 내) | 같은 번호 2회 이상 등장 | 첫 번째 행만 유효, 이후 행 에러: "파일 내 중복" |
| 전화번호 중복(기존 DB) | 이미 이 회사에 등록된 번호 | 에러: "이미 등록된 번호" (건너뜀) |
| 행 수 | 최대 500행 | 초과 시 파일 자체 반려, 안내 메시지 표시 |

### 3.2 파일 형식 검증

- 헤더가 `부서, 이름, 사번, 전화번호` 순서/명칭과 다르면 → 업로드 자체를 반려하고 "양식을 다운로드해서 사용해주세요" 안내
- `.xlsx`, `.csv` 외 확장자 업로드 시 반려

### 3.3 사번 자동 채번 규칙

- 사번이 비어있는 행: 서버가 자동 생성 (`{company_id 접두어}-{순번}` 형식, 예: `DTC-0049`)
- 파일 내 사번이 기존 DB와 중복되면 → 에러 처리 (사번은 입력 시엔 유일해야 함, 자동채번은 항상 유일하게 생성)

## 4. 미리보기 화면 (Step 3)

```
┌──────────────────────────────────────────────┐
│  업로드 결과 확인                        ✕    │
├──────────────────────────────────────────────┤
│  ✅ 정상 48건    ⚠️ 에러 3건                   │
│                                                │
│  [정상 48건 리스트 (기본 펼침)]                │
│   부서 | 이름 | 사번 | 전화번호                │
│   ...                                          │
│                                                │
│  [에러 3건 리스트 (빨간 배경, 사유 표시)]       │
│   3행: 전화번호 형식 오류 (010-abcd-5678)      │
│   12행: 이름 누락                              │
│   27행: 이미 등록된 번호 (중복)                │
├──────────────────────────────────────────────┤
│  [이전]              [정상 48건만 등록 확정]   │
└──────────────────────────────────────────────┘
```

- 에러 있는 행은 등록 대상에서 자동 제외, 정상 행만 확정 가능
- 에러 0건이면 "정상 N건 등록 확정" 버튼만 강조 표시
- 에러 건은 CSV로 재다운로드 가능하게 `[에러 목록 다운로드]` 버튼 제공 (관리자가 수정해서 재업로드하기 편하도록)
- [등록 확정] 클릭 시에만 실제 DB insert 수행

## 5. 데이터 모델 반영

```
accounts                             -- 기존 테이블에 컬럼 추가/활용
  - id
  - phone                 -- 필수, 로그인 식별자
  - name                  -- 필수
  - account_type: 'ledger'   -- 일괄등록은 항상 ledger 계정만 생성
  - company_id             -- 업체관리자 소속 회사로 고정
  - department             -- 신규 컬럼, nullable
  - employee_no             -- 신규 컬럼, nullable → 비어있으면 자동채번
  - status: 'invited' | 'active' | 'inactive'
    -- 일괄등록 직후 기본값 'invited'
    -- 해당 전화번호로 직원이 앱에서 최초 로그인/인증하면 'active'로 전환
```

## 6. API 스펙

```
GET  /admin/employees/template
  → 엑셀 양식 파일 다운로드 (부서,이름,사번,전화번호 헤더 + 예시행)

POST /admin/employees/bulk-upload/parse
  Request: multipart file (xlsx/csv)
  → DB 저장 없이 파싱+검증만 수행
  Response: {
    valid: [ { row, department, name, employee_no, phone } ],
    errors: [ { row, reason } ]
  }

POST /admin/employees/bulk-upload/confirm
  Request: { valid_rows: [...] }   -- 미리보기에서 최종 확인된 정상 행만 전달
  → accounts 레코드 생성 (status='invited', account_type='ledger')
  Response: { created_count }
```

- `parse`와 `confirm`을 분리해서, 미리보기 단계에서는 절대 DB에 쓰지 않도록 함 (실수로 잘못된 파일 업로드해도 되돌리기 쉬움)

## 7. 구현 순서 (마일스톤)

1. **M1**: `accounts` 테이블에 `department`, `employee_no`, `status='invited'` 반영 (마이그레이션)
2. **M2**: 양식 다운로드 API + 팝업 모달 골격 (Step1~2 업로드 UI)
3. **M3**: 파싱/검증 API (`parse`) + 미리보기 화면 (Step3, 정상/에러 구분 표시)
4. **M4**: 확정 API (`confirm`) + 에러 목록 다운로드 기능
5. **M5**: 직원 최초 로그인 시 전화번호 매칭 → `invited` → `active` 자동 전환 로직

## 8. 완료 기준 (Acceptance Criteria)

- [ ] 양식과 다른 헤더로 업로드하면 반려되고 안내 메시지가 뜬다
- [ ] 이름/전화번호 누락 행은 에러로 분류되고, 정상 행만 등록 확정 가능하다
- [ ] 파일 내 중복 전화번호, 기존 등록된 전화번호가 각각 올바른 사유로 에러 처리된다
- [ ] 사번을 비워둔 행은 자동으로 유일한 사번이 채번된다
- [ ] [등록 확정] 누르기 전까지는 어떤 데이터도 DB에 저장되지 않는다
- [ ] 등록된 계정은 `status='invited'`로 생성되고, 직원이 실제 로그인하면 `active`로 전환된다
- [ ] 500행 초과 파일은 업로드 자체가 반려된다

## 9. 금지 사항 / 주의

- 미리보기(파싱) 단계에서 DB에 직접 insert하는 로직 금지 — 반드시 confirm 단계에서만 실제 저장
- 에러 행을 무시하고 전체 파일을 일괄 등록 처리하는 로직 금지 — 정상 행만 선별 등록
- 전화번호 중복 검증 없이 등록하는 로직 금지 (한 회사 내 동일 전화번호 계정 중복 생성 방지)
