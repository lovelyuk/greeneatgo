# 식당관리자 웹 사용설명서

## 1. 식당관리자 웹이란?

식당관리자 웹은 식당 운영자가 다음 업무를 처리하는 화면입니다.

- 매장 결제 QR 확인·인쇄
- 회사 생성 및 담당자 초대
- 업체별 거래 조회와 정산
- 일반 고객용 식권 패키지 관리
- 회사 직원용 장부 상품 관리
- 앱 푸시 알림 및 공지사항 발행
- 고객 리뷰 확인과 사장님 답글
- 오늘의 뷔페 메뉴 관리

관리자 웹 주소: `https://greeneatgo.vercel.app/`

---

## 2. 로그인과 공통 기능

### 로그인

1. 이메일과 비밀번호를 입력합니다.
2. **운영 시작하기**를 누릅니다.

처음 사용하는 경우 플랫폼 운영자에게 받은 초대링크에서 이름과 비밀번호를 등록해 초대를 수락합니다.

### 헤더

- **새로고침**: 모든 탭의 최신 데이터를 다시 불러옵니다.
- **로그아웃**: 관리자 계정에서 나갑니다.
- 우측 하단 이메일 옆 **톱니바퀴**: 관리자 이름 또는 비밀번호를 변경합니다.

모바일에서는 상단 버튼이 아이콘으로 표시되고, 메뉴 탭은 화면 하단 내비게이션으로 표시됩니다.

---

## 3. 메뉴 구성

| 메뉴 | 주요 기능 |
|---|---|
| 메인 | 현황, 오늘 결제 알림, 매장 QR |
| 업체 관리 | 회사 생성·초대, 계약, 거래·정산 |
| 판매 상품(일반) | 일반 고객용 식권 패키지 |
| 식당 상품(장부) | 회사 직원 QR 결제용 메뉴 |
| 알림 | 사용자 앱 푸시 발송 |
| 공지사항 | 공지 게시·고정·숨김 |
| 리뷰 | 고객 리뷰·답글·노출 관리 |
| 오늘 뷔페 메뉴 | 날짜별 메뉴와 이미지 저장 |

---

## 4. 메인

### 현황 카드

- 판매 중인 장부 상품 수
- 연결된 장부업체 수
- 전체 거래 건수

### 오늘의 결제 알림

오늘 승인된 최근 결제 최대 10건을 표시합니다.

- 결제 시각
- 직원 이름
- 결제 구분: 장부, 보조금, 식권
- 금액

새 거래가 들어오면 알림음이 재생되고 목록이 갱신됩니다. 다른 탭을 보고 있을 때는 메인 탭에 미확인 건수 배지가 표시됩니다.

### 내 매장 결제 QR

- 매장 QR과 결제 링크 확인
- **링크 복사**
- **PDF 다운로드**: 인쇄용 화면을 열어 브라우저의 PDF 저장 또는 인쇄 기능 사용

브라우저가 새 창을 차단하면 PDF 기능이 열리지 않을 수 있습니다. 팝업을 허용한 후 다시 시도하세요.

---

## 5. 업체 관리

### 신규 업체 생성 및 이메일 초대

1. 신규 회사명을 입력합니다.
2. 담당자 이메일을 입력합니다.
3. 필요하면 담당자 연락처를 입력합니다.
4. **신규 생성 + 이메일 초대**를 누릅니다.

회사가 생성되고 담당자에게 업체관리자 초대 이메일이 발송됩니다. 이메일 발송에 실패하더라도 회사와 초대 정보가 생성됐을 수 있으므로 화면의 결과 메시지를 확인하세요.

### 연결 업체 목록

목록에서 다음 정보를 확인할 수 있습니다.

- 회사명
- 담당자 이메일
- 회사·연결 상태
- 거래 건수
- 계약 상태
- 초대 상태
- 이메일 발송 상태

대기 중인 초대는 **링크**로 주소를 확인·복사하거나 **재전송**할 수 있습니다.

### 계약 설정

업체 행에서 계약 설정을 열어 다음 내용을 입력합니다.

- 정산 주기: 월말 또는 매월 특정일
- 특정 정산일
- 계약 단가

> 보조금 사용 여부와 회사·식당 지원액 입력 UI가 표시될 수 있지만, 현재 운영 API의 저장 연결 상태를 먼저 확인해야 합니다. 보조금 계약을 실제 적용하기 전에는 저장 후 다시 열어 값이 유지되는지 반드시 검증하세요.

### 업체 거래내역

업체 행의 **N건 보기**를 누릅니다.

#### 조회 기능

- 시작일·종료일 선택
- 부서·이름·사번 검색
- 총 이용금액과 이용건수
- 취소 건수
- 미정산 잔액
- 거래별 날짜, 직원, 메뉴, 구분, 청구액

시작일은 종료일보다 늦을 수 없습니다.

#### 내보내기

- **엑셀 다운로드**
- **PDF 청구서**

#### 정산 생성

1. 정산할 시작일과 종료일을 선택합니다.
2. 거래와 금액을 확인합니다.
3. **선택 기간 정산 생성**을 누릅니다.
4. 정산이력 탭에서 생성된 회차를 확인합니다.

중복 기간 정산은 만들 수 없습니다.

#### 입금 확인

1. 정산이력 탭에서 입금대기 또는 연체 회차를 찾습니다.
2. **입금확인**을 누릅니다.
3. 입금일을 입력합니다.
4. 상태가 입금완료로 변경됐는지 확인합니다.

## 6. 판매 상품(일반)

일반 사용자가 앱에서 카드로 구매하는 식권 패키지를 관리합니다.

### 상품 등록 항목

- 패키지명
- 기본 장수
- 보너스 장수
- 장당 정가
- 할인율
- 판매중·숨김 상태
- 노출순서
- 이벤트 상품 여부와 기간
- 상품 이미지

### 등록 순서

1. 패키지명과 장수를 입력합니다.
2. 정가와 할인율을 입력합니다.
3. 판매가·총 지급 장수·실질 장당 금액을 확인합니다.
4. 이미지를 선택하고 정사각형으로 위치와 확대를 조정합니다.
5. **상품 등록**을 누릅니다.

보너스와 할인을 동시에 적용하면 추가 확인이 표시됩니다. 이벤트 종료일은 시작일보다 늦게 설정하세요.

### 기존 상품 관리

- **수정**: 상품 정보 변경
- **숨김**: 사용자 앱에서 노출 중단
- **판매 재개**: 다시 노출

삭제 기능은 제공되지 않습니다.

---

## 7. 식당 상품(장부)

회사 직원이 매장 QR로 장부 결제할 때 사용하는 식당 메뉴를 관리합니다.

### 신규 등록

1. 상품명을 입력합니다.
2. 가격을 입력합니다.
3. 카테고리를 입력합니다.
4. 필요하면 상품 이미지를 선택하고 정사각형으로 조정합니다.
5. 등록 버튼을 누릅니다.

### 기존 상품 관리

- 이미지 변경
- **숨김**: 직원 앱 상품 목록에서 제외
- **판매중**: 다시 노출

현재 화면에서는 기존 상품의 이름·가격·카테고리를 직접 편집하거나 삭제하는 기능은 제공되지 않습니다.

---

## 8. 알림

사용자 앱으로 푸시 알림을 발송합니다.

### 발송 대상

- 전체 사용자: 회사 직원과 일반 사용자
- 일반 사용자만: 개인 식권 구매 사용자

### 발송 순서

1. 대상을 선택합니다.
2. 발송 가능 사용자와 등록 기기 수를 확인합니다.
3. 제목과 내용을 입력합니다.
4. 필요하면 **미리보기**를 확인합니다.
5. **발송하기**를 누릅니다.
6. 확인 창에서 대상과 내용을 다시 확인합니다.
7. **공지 발송**을 누릅니다.

발송 시작 후에는 취소할 수 없습니다. 발송 가능 사용자 수와 기기 수는 서로 다를 수 있습니다.

### 발송 이력

**발송 이력**에서 날짜, 대상, 제목, 사용자 접수 수, 기기 성공·실패 수를 확인합니다. 성공 수는 실제 열람 수가 아니라 푸시 서버가 접수한 수입니다.

> 현재 대상 산정은 특정 계약 회사만이 아니라 서비스의 활성 직원·일반 사용자 전체가 될 수 있습니다. 대량 발송 전 대상 표시를 반드시 확인하세요.

---

## 9. 공지사항

### 공지 게시

1. 제목과 내용을 입력합니다.
2. 필요하면 **상단 고정**을 선택합니다.
3. 앱 푸시도 보낼 경우 **푸시 알림도 함께 발송**을 선택합니다.
4. **게시하기**를 누릅니다.

### 게시 후 관리

- **상단 고정 / 고정 해제**
- **숨김 / 노출로 복원**

현재 화면에서는 게시된 공지의 제목·내용 직접 수정이나 삭제 기능은 제공되지 않습니다.

---

## 10. 리뷰

고객이 구매·이용을 인증해 작성한 리뷰를 관리합니다.

### 확인 기능

- 평균 별점
- 노출 중인 리뷰 수
- 최신순 또는 낮은 별점순 정렬
- 별점, 내용, 첨부 이미지, 작성 시각

### 사장님 답글

1. 리뷰 아래 답글을 입력합니다.
2. **답글 저장**을 누릅니다.

### 노출 관리

- **숨김 처리**
- **노출로 복원**

리뷰를 직접 삭제하는 기능은 없습니다.

---

## 11. 오늘 뷔페 메뉴

오늘 또는 앞으로 제공할 메뉴를 날짜별로 저장합니다.

1. 메뉴 날짜를 선택합니다.
2. 제목을 입력합니다.
3. 메뉴 설명을 입력합니다.
4. 필요하면 최대 5MB의 이미지를 선택합니다.
5. **선택 날짜 메뉴 저장**을 누릅니다.

저장된 일정의 **수정**을 누르면 해당 날짜 메뉴가 입력 폼에 다시 표시됩니다. 과거 날짜는 저장할 수 없고, 일정 목록에는 오늘 이후 메뉴가 표시됩니다.

---

## 12. 권장 일일 운영 순서

1. 로그인 후 메인 결제 알림과 현황 확인
2. 오늘 뷔페 메뉴 확인·수정
3. 상품의 판매 상태와 가격 확인
4. 신규 업체 초대 및 초대 상태 확인
5. 업체별 거래와 미정산 금액 확인
6. 정산 기간 생성 또는 입금 확인
7. 신규 리뷰 확인 및 답글 작성
8. 필요 시 공지·푸시 발송
9. 업무 종료 후 로그아웃

---

## 13. 자주 묻는 질문

### 새 결제가 메인에 안 보여요

- **새로고침**을 누르세요.
- 오늘 승인된 거래인지 확인하세요.
- 현재 로그인 계정이 해당 식당에 연결돼 있는지 확인하세요.

### 상품을 등록했는데 앱에 안 보여요

- 상품이 판매중인지 확인하세요.
- 이벤트 기간이 시작됐는지 확인하세요.
- 앱이 바라보는 운영 식당과 현재 관리 식당이 같은지 확인하세요.
- 앱에서 새로고침하거나 다시 로그인하세요.

### 초대 이메일이 도착하지 않아요

- 이메일 주소를 확인하세요.
- 스팸함을 확인하세요.
- 목록의 **링크**로 초대주소를 직접 전달하거나 **재전송**하세요.

### 기능이 비활성화돼요

상품 이벤트, 알림, 오늘 메뉴, 기간 정산 등은 운영 DB 적용 상태에 따라 비활성화될 수 있습니다. 화면의 안내 문구를 확인하고 시스템 관리자에게 문의하세요.
